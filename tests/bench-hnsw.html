HNSW bench — Euclidean vs Hyperbolic (P3.A)

500 synthetic 512-d vectors, 100 random queries, recall@5 measured against a brute-force top-5 under each index's own metric. Latency is per .search() call. Both indices use the crate default ef_search = 50.

Two scenarios, one gate. The PASS/FAIL summary below reflects only the spherical run (L2-normalised unit vectors, matching our actual CnnEmbedder output). The ball-native run uses random points inside the Poincaré ball — the distribution hyperbolic HNSW was designed for — and is shown alongside to make the geometry-vs-data-manifold trade-off concrete. If hyperbolic loses on spherical but wins (or closes the gap) on ball-native, the takeaway is "we're feeding the wrong manifold," not "the index is broken."
